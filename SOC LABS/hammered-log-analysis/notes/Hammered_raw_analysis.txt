 Case: Hammered — CyberDefenders

## Initial Log Review

## user.log

Reviewed `user.log` for correlated user activity associated with the suspicious events.

No relevant indicators or anomalous behavior were identified during the reviewed timeframe.



## auth.log Findings

Multiple failed SSH authentication attempts targeting privileged accounts were identified from external IP addresses.

Several successful SSH authentications were also observed shortly after failed authentication attempts from the same source hosts.

Additional privileged activity included:

* `sudo` usage
* `su` usage
* root password modifications
* privileged account creation
* remote root authentications

The observed behavior is consistent with unauthorized privileged access and likely persistence establishment.

---

## Apache Weblog Findings

Apache weblog analysis identified:

* anomalous HTTP requests associated with proxy validation/scanning behavior
* repeated access attempts to administrative resources such as `/wp-admin/`
* multiple instances of privileged Apache service manipulation activity performed by `user1`

A significant portion of observed `/feed/` polling traffic and WordPress cron activity appeared consistent with legitimate automated behavior.

# Assessment

The investigation identified multiple suspicious SSH authentication events originating from external IP addresses. Several successful authentications occurred shortly after failed login attempts and reverse DNS inconsistencies flagged by `sshd` as potential break-in attempts.

Most notably, a successful remote SSH authentication as `root` was followed by the creation of a new account (`packet`) configured with `UID=0`, effectively granting root-equivalent privileges. Password assignment activity for the newly created account strongly indicates persistence establishment following privileged access.

Additional findings included:

* privileged Apache service manipulation activity
* repeated authentication attempts targeting privileged accounts
* root password modification events
* anomalous web requests associated with proxy validation/scanning behavior

While some observed Apache activity appeared legitimate (RSS feed polling and WordPress cron operations), the overall sequence of authentication anomalies, privileged activity, and persistence-related actions strongly suggests unauthorized system compromise.

---

# Severity Assessment

## Severity: High

### Justification

* Successful remote authentication as `root`
* Creation of additional `UID=0` privileged account
* Root password modification activity
* Repeated suspicious SSH authentication attempts from external IPs
* Privileged manipulation of Apache services
* Indicators consistent with persistence establishment

---

# Scope and Limitations

The investigation was limited to the log sources available within the lab environment.

No direct evidence of:

* malware deployment
* data exfiltration
* webshell installation
* lateral movement

was identified during the reviewed timeframe.

However, the available evidence strongly supports unauthorized privileged access and persistence activity on the affected host.

---

# Recommended Action

Escalation recommended for:

* potential host compromise
* unauthorized privileged persistence
* root-level account manipulation
* further forensic validation and containment activities
